PhotonBolt Wallet Submission Notes

This file contains submission-ready text for the PhotonBolt Wallet extension, aligned to the current manifest and extension behavior.

Manifest permissions: storage, activeTab, scripting, <all_urls>

Single Purpose

Recommended text

PhotonBolt Wallet is a browser wallet extension for connecting to PhotonBolt-enabled websites and allowing users to view wallet balances, connect accounts, approve Bitcoin and RGB asset actions, sign messages, and send transactions from their wallet.

Short Description

Connect to PhotonBolt-enabled apps, view balances, approve Bitcoin and RGB asset actions, sign messages, and send transactions from your wallet.

Detailed Description

PhotonBolt Wallet is a browser extension that injects a wallet provider into compatible websites so users can securely connect their wallet and approve requested actions. Users can: - connect supported dApps to their wallet - view wallet address and balances - approve Bitcoin and RGB asset transactions - sign messages when requested - manage transaction approvals inside the extension PhotonBolt Wallet does not exist to browse the web, scrape content, inject ads, or modify pages for unrelated purposes. Its only purpose is to provide a user-controlled wallet connection and approval interface for PhotonBolt-compatible web applications.

Permission Justifications

Permission Why it is needed
storage Used to store wallet state, approved site connections, selected network, wallet settings, and extension session data needed for the wallet to function consistently across tabs and browser restarts.
activeTab Used to interact only with the currently active tab when a site requests a wallet action and the extension needs to send approval or connection updates back to that page.
scripting Used to inject the PhotonBolt provider bridge into web pages so compatible dApps can communicate with the wallet extension.
<all_urls> Required because the wallet provider must be available on any website a user chooses to connect to. The extension does not use this access for unrelated browsing activity; it uses it only to inject the wallet bridge and relay wallet requests on user-visited pages.

Host Permission Explanation

PhotonBolt Wallet requests host access on all URLs because wallet-enabled websites can be hosted on any domain. The extension injects a provider script and content bridge only so websites chosen by the user can request wallet connection and wallet actions. This host access is not used for advertising, analytics scraping, page manipulation unrelated to wallet use, or background monitoring of browsing activity.

Data Use Disclosure Draft

What user data the extension handles

  • Wallet addresses and balances shown to the user.
  • Approved site connection records stored locally.
  • Transaction request payloads, message-signing requests, and approval decisions.
  • Wallet configuration and selected network settings.

What the extension does not do

  • It does not sell browsing data or personal data.
  • It does not use host access for advertising, profiling, or unrelated page modification.
  • It does not collect passwords from websites.

Suggested disclosure text

PhotonBolt Wallet processes wallet connection requests, account addresses, balances, transaction approvals, and signing requests so users can interact with PhotonBolt-compatible applications. Data is used only to operate the wallet functionality requested by the user. The extension does not sell personal information or use page access for unrelated advertising or tracking purposes.

Privacy Policy Summary For Review

PhotonBolt Wallet stores wallet-related settings and approved site connections locally in the browser. When users connect to a website, the extension exchanges wallet request and response messages with that site so the requested wallet function can be completed. Wallet data is used only to provide wallet features such as connection, balance retrieval, transaction approval, and message signing. The extension does not use broad host access for unrelated browsing surveillance or ad targeting.

If you publish a full external privacy policy URL, make sure it matches the extension behavior and any backend services the wallet calls.

Reviewer Notes

Reviewer guidance: - The extension injects a wallet provider on pages so dApps can call the PhotonBolt wallet API. - A content script relays requests between the page and the extension background worker. - Approval flows are shown to the user before sensitive actions such as connect, transaction approval, or signing are completed. - The extension's purpose is limited to wallet connectivity and wallet approvals for Bitcoin and RGB asset activity.
