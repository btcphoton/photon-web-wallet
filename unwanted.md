# Unwanted Files for Submission
The following files and directories are typically not needed for a clean Chrome‑extension submission and should be moved to a new folder `unwanted_for_submission` at the project root:

- `.env` – contains local secrets and environment variables
- `.env.example` – example env file, not required for submission
- `.git` – version‑control metadata
- `node_modules/` – local npm packages (should be regenerated via `npm install`)
- `dist/` – build output (will be regenerated by the build script)
- `checkin.sh` – development helper script
- `gpush.sh` – deployment helper script
- `generate_logo.py` – logo generation script
- `process_logo.py` – logo processing script
- `HelperText.txt` – internal documentation not needed for reviewers
- `ChromeWebStore.html` – Chrome Web Store publishing page, not required for source submission
- `animation.html` – demo animation page, not part of the extension bundle
- `test-dapp.html` – test page for DApp integration
- `test-testnet4.js` – test script for testnet interactions
- `test-dapp.html` and `test-testnet4.js` (both under root) – test assets
- `scripts/` (if it only contains dev scripts) – optional, move if not needed
- `docs/` (if it contains internal docs) – optional, move if not required for reviewers

You can review this list and let me know if any items should be kept. Once approved, I will create the `unwanted_for_submission` folder and move the listed items there.
